共有カレンダーWebアプリ 設計書（初版）

1. システム概要

目的:複数ユーザーで共有できる「カレンダー形式」のWebアプリ。各ユーザーが以下を登録できる：

予定（schedule）

休み（off）

メモ（memo）

各イベントごとに

内容（タイトル・詳細・タグ・色）

公開範囲（共有する／しない／一部ユーザーのみ） を設定可能

2. 技術スタック

2.1 フロントエンド（モダン構成）

Next.js（React + TypeScript）

UIライブラリ：MUI または Tailwind CSS

カレンダーUI：FullCalendar（React版）

状態管理：React Query or Zustand

認証：JWT + Cookie（HTTP Only）

2.2 バックエンド

Node.js + NestJS（TypeScript）

もしくは Express でも可（ここでは NestJS 前提で記述）

API形式：REST（将来GraphQL拡張も可）

認証：JWT（アクセストークン＋リフレッシュトークン）

2.3 データベース（Web上で稼働）

PostgreSQL（Supabase / Railway / Render などのマネージドDB）

マイグレーション：Prisma or TypeORM

2.4 ホスティング

フロント：Vercel

バックエンド：Railway / Render

DB：Supabase（PostgreSQL）

6. API設計（REST）

6.1 認証

POST /auth/register

入力：name, email, password

出力：ユーザー情報＋トークン

POST /auth/login

入力：email, password

出力：ユーザー情報＋トークン

6.2 カレンダー

GET /calendars

自分がメンバーのカレンダー一覧

POST /calendars

カレンダー作成（owner = 自分）

GET /calendars/:id/membersPOST /calendars/:id/members

6.3 イベント

GET /calendars/:id/events?month=2026-06

返却ルール：

visibility = public → 常に返す

visibility = private → user_id = 自分 のみ返す

visibility = custom → event_shared_users に自分が含まれる場合のみ返す

POST /events

{
  "calendar_id": "uuid",
  "date": "2026-06-10",
  "type": "memo",
  "title": "買い物メモ",
  "memo": "牛乳・卵",
  "tag": "家事",
  "color": "#FFAA00",
  "visibility": "custom",
  "shared_user_ids": ["user-uuid-1", "user-uuid-2"]
}

PUT /events/:idDELETE /events/:id
